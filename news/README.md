# Структура раздзела "Навіны"

Гэты раздзел складаецца з дзвюх частак, якія трэба весці разам уручную,
пакуль не зроблены Этап 2 (генератар старонак):

| Markdown-крыніца (для CMS)              | Гатовая старонка (для сайта)     |
|------------------------------------------|------------------------------------|
| `content/news/2026-06-25-nabor-2026.md`  | `news/nabor-2026.html`         |
| `content/news/2026-06-20-vypusk-2026.md` | `news/vypusk-2026.html`        |

Дата ў пачатку імя файла — толькі для храналагічнага сартавання ў CMS,
астатняя частка імя ("nabor-2026", "vypusk-2026") аднолькавая ў абодвух
файлах — гэта і ёсць сувязь паміж крыніцай і гатовай старонкай.

## Як дадаць новую навіну (да Этапу 2)

1. Стварыць новы файл у `content/news/` праз Decap CMS (або ўручную) —
   гэта застанецца "крыніцай праўды" для тэксту.
2. Уручную стварыць адпаведную старонку `news/назва-навіны.html` па ўзоры
   існуючых файлаў у гэтай папцы (скапіяваць структуру, уставіць тэкст).
3. Дадаць новую карцінку-картку ў спіс на галоўнай старонцы навін
   (`news.html`), са спасылкай на новую старонку.

## Пасля Этапу 2

Калі будзе наладжаны генератар (напр. Eleventy), пункты 2 і 3 будуць
адбывацца аўтаматычна пры кожным каміце ў `content/news/` — гэты файл
можна будзе выдаліць, бо CMS і сайт будуць злучаны напрамую.
